👥 Assigning Roles — BRANCH_USERS Map

Open this HTML file and locate the BRANCH_USERS object in the <script> section.
Add each Firebase UID with the correct role:

const BRANCH_USERS = {
  // Branch Accountants — one per branch
  "UID_HQ_ACCOUNTANT": { branch:"HQ", role:"accountant" },
  "UID_B1_ACCOUNTANT": { branch:"B1", role:"accountant" },
  // … add remaining branches …

  // Regional Accountant — views all, no write access
  "UID_REG_ACCOUNTANT": { branch:"REGIONAL", role:"regional_accountant" },

  // Regional Overseer — sole administrator
  "UID_OVERSEER": { branch:"REGIONAL", role:"regional_overseer" },
};

⚠️ Only one Regional Overseer account should exist. This account cannot be shared. Get each Firebase UID from: Firebase Console → Authentication → Users → copy the UID column.

Firebase Setup Guide

Email/Password auth + Firestore real-time data

🔥 How to Connect Firebase (Step by Step)

1

Create Firebase Project

Go to console.firebase.google.com → "Add project" → Name it mfm-akure-portal → Disable Google Analytics → Create project.

2

Enable Firebase Authentication

In your project → Authentication → Get started → Email/Password provider → Enable → Save.

3

Create All User Accounts

Authentication → Users → Add user for each person. Suggested email format:

[email protected] — HQ Branch Accountant
[email protected] — Unity Zone Accountant
[email protected] — Ilere Zone Accountant
[email protected] — Irese 1 Accountant
[email protected] — Irese 2 Accountant
[email protected] — Branch 1 Accountant
[email protected] — Branch 2 Accountant
[email protected] — Regional Accountant (view-only all branches)
[email protected] — Regional Overseer (full administrator)

After creating each user, copy the UID shown in the Users table — you will need it in Step 7.

4

Get Your Web App Config

Project Settings → click the </> web icon → Register app as mfm-portal → Copy the firebaseConfig object.

5

Paste Config into This File

Open this HTML file in Notepad++ → Find firebaseConfig in the <script type="module"> block → Replace the placeholder values with your real config.

6

Enable Firestore Database

Firebase Console → Firestore Database → Create database → Production mode → Region: europe-west1 → Enable.

7

Update BRANCH_USERS Map

In this file find the BRANCH_USERS object and fill in each Firebase UID with the correct branch and role:

"PASTE_UID_HERE": { branch:"HQ", role:"accountant" }
"PASTE_UID_HERE": { branch:"REGIONAL", role:"regional_accountant" }
"PASTE_UID_HERE": { branch:"REGIONAL", role:"regional_overseer" }
8

Set Firestore Security Rules

rules_version = '2';
service cloud.firestore {
 match /databases/{database}/documents {
  match /transactions/{doc} {
   allow read, write: if request.auth != null;
  }
 }
}

Click Publish.

9

Add Authorised Domain

Authentication → Settings → Authorised domains → Add your hosting domain, e.g. yourusername.github.io → Add.

✅ Local Mode — Works Without Firebase

Without Firebase the portal uses browser localStorage with a simple email/username fallback for testing. Once Firebase config and UID map are in place it automatically switches to Firebase Auth + Firestore cloud mode.

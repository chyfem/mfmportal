Firebase Setup Guide

Connect for real-time multi-branch data

🔥 How to Connect Firebase (Step by Step)

1

Create Firebase Project

Go to console.firebase.google.com → "Add project" → Name it mfm-akure-portal → Disable Google Analytics → Create project.

2

Get Your Config Credentials

Dashboard → click the </> web icon → Register app as mfm-portal → Copy the firebaseConfig object shown.

3

Paste Credentials into This File

Open this HTML file in Notepad++ → Press Ctrl+F → Search for PASTE_YOUR_FIREBASE_CONFIG_HERE → Replace the entire placeholder object with your real config.

4

Enable Firestore Database

Firebase Console → Firestore Database → Create database → Production mode → Region: europe-west1 → Enable.

5

Set Firestore Rules

Firestore → Rules tab → Replace all content with:

rules_version = '2';
service cloud.firestore {
 match /databases/{database}/documents {
  match /transactions/{doc} {
   allow read, write: if request.auth != null;
  }
 }
}

Click Publish.

6

Enable Firebase Authentication

Authentication → Get started → Email/Password → Enable → Save.

7

Create Branch User Accounts

Authentication → Users → Add user for each branch:

hq@mfmararomi.orgokeijebu@mfmararomi.orgijoka@mfmararomi.orgshagari@mfmararomi.orgaswani@mfmararomi.orgobaile@mfmararomi.orgijapo@mfmararomi.orgadmin@mfmararomi.org
8

Add GitHub Pages Domain to Firebase

Authentication → Settings → Authorised domains → Add domain → type your GitHub Pages URL e.g. yourusername.github.io → Add.

9

Update Branch Emails in the HTML

In the HTML file search for BRANCH_USERS and update the email addresses to match exactly what you created in Firebase Authentication.

✅ Right Now — Portal Works in Local Mode

The portal currently runs in local storage mode — all data is saved in your browser and works fully offline. Once you add Firebase credentials, it automatically switches to cloud mode where all branches share data in real time. Your local data will not be lost during the switch.
